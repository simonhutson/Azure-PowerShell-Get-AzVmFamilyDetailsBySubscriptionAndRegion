# Azure VM Family Details by Subscription and Region

PowerShell scripts for listing Azure VM families available to a specific subscription in a specific Azure region. The main script groups matching VM SKUs into VM family or series rows and includes generation support, CPU architecture, disk controller type, local temporary disk presence, deployment restrictions, retirement status, and the matching SKU names.

## Scripts

- `Get-AzVmFamilyDetailsBySubscriptionAndRegion.ps1` - Queries Azure Compute SKU metadata for a subscription and region, filters it, and prints a family-level table.
- `Update-RetiredAzVmSizesFile.ps1` - Downloads the retired Azure VM size series list from Microsoft Learn and writes `retired-vm-sizes.json`.
- `retired-vm-sizes.json` - Optional retirement data file consumed by the main script.

## Prerequisites

- PowerShell 7 or Windows PowerShell.
- Azure PowerShell with the Az.Compute and Az.Accounts cmdlets available.
- An authenticated Azure PowerShell session.
- Access to the target Azure subscription.

Install the Az module if needed:

```powershell
Install-Module Az -Scope CurrentUser
```

Sign in before running the query script:

```powershell
Connect-AzAccount
```

## Usage

Query all Gen1 or Gen2 capable VM families in a region:

```powershell
.\Get-AzVmFamilyDetailsBySubscriptionAndRegion.ps1 `
    -SubscriptionId 00000000-0000-0000-0000-000000000000 `
    -Location uksouth
```

Filter to SCSI-capable families with a local temporary/resource disk:

```powershell
.\Get-AzVmFamilyDetailsBySubscriptionAndRegion.ps1 `
    -SubscriptionId 00000000-0000-0000-0000-000000000000 `
    -Location uksouth `
    -DiskInterface SCSI `
    -LocalTemporaryDisk Required
```

Filter to families without a local temporary/resource disk:

```powershell
.\Get-AzVmFamilyDetailsBySubscriptionAndRegion.ps1 `
    -SubscriptionId 00000000-0000-0000-0000-000000000000 `
    -Location westeurope `
    -LocalTemporaryDisk Excluded
```

Filter to ARM-based families:

```powershell
.\Get-AzVmFamilyDetailsBySubscriptionAndRegion.ps1 `
    -SubscriptionId 00000000-0000-0000-0000-000000000000 `
    -Location uksouth `
    -CPUArchitecture ARM
```

Filter to Intel and AMD families (multi-select, comma-separated):

```powershell
.\Get-AzVmFamilyDetailsBySubscriptionAndRegion.ps1 `
    -SubscriptionId 00000000-0000-0000-0000-000000000000 `
    -Location uksouth `
    -CPUArchitecture Intel,AMD
```

Show only families that are not retired or announced for retirement:

```powershell
.\Get-AzVmFamilyDetailsBySubscriptionAndRegion.ps1 `
    -SubscriptionId 00000000-0000-0000-0000-000000000000 `
    -Location uksouth `
    -RetirementStatus Available
```

Combine filters, for example available AMD or ARM families that have a local temporary disk:

```powershell
.\Get-AzVmFamilyDetailsBySubscriptionAndRegion.ps1 `
    -SubscriptionId 00000000-0000-0000-0000-000000000000 `
    -Location uksouth `
    -CPUArchitecture AMD,ARM `
    -RetirementStatus Available `
    -LocalTemporaryDisk Required
```

Use the refreshed retirement data file:

```powershell
.\Get-AzVmFamilyDetailsBySubscriptionAndRegion.ps1 `
    -SubscriptionId 00000000-0000-0000-0000-000000000000 `
    -Location uksouth `
    -RetiredSizesPath .\retired-vm-sizes.json
```

## Parameters

| Parameter | Required | Default | Description |
| --- | --- | --- | --- |
| `SubscriptionId` | Yes | None | Azure subscription ID to query. Must be a valid GUID. |
| `Location` | Yes | None | Azure region to query, for example `uksouth`, `westeurope`, or `eastus`. The alias `Region` is also supported. |
| `DiskInterface` | No | `Any` | Use `Any`, `SCSI`, or `NVMe` to filter by disk controller support. |
| `LocalTemporaryDisk` | No | `Any` | Use `Any`, `Required`, or `Excluded` to filter by local temporary/resource disk presence. |
| `CPUArchitecture` | No | `Any` | One or more of `Any`, `Intel`, `AMD`, or `ARM` (comma-separated) to filter by CPU architecture. A SKU is included if it matches any of the supplied values. |
| `RetirementStatus` | No | `Any` | Use `Any`, `Available`, `Retirement Announced`, or `Retired` to filter by retirement status. |
| `RetiredSizesPath` | No | Embedded fallback list | Path to a JSON file generated by `Update-RetiredAzVmSizesFile.ps1`. |

## Output

The main script writes a table with one row per VM family or series.

| Column | Description |
| --- | --- |
| `VMFamily` | Normalized VM family or series name. |
| `CPUArchitecture` | `Intel`, `AMD`, or `ARM`, derived from SKU metadata and Azure VM size naming. |
| `HyperVGenerations` | Hyper-V generation support, shown as `Gen1`, `Gen2`, or both. |
| `DiskControllerTypes` | Disk controller types reported by Azure, or `SCSI (inferred)` when the capability is absent and treated as legacy SCSI-class. |
| `HasLocalTemporaryDisk` | Whether the family has a local temporary/resource disk, derived from `MaxResourceVolumeMB`. |
| `RetirementStatus` | `Available`, `Retirement Announced`, or `Retired`. |
| `RegionalDeploymentRestrictions` | Whether regional deployment is restricted. |
| `ZonalDeploymentRestrictions` | Whether zonal deployment is restricted, including specific zones where available. |
| `Skus` | Matching Azure VM SKU names in the family, ordered from smaller to larger vCPU counts. |

## Refresh Retirement Data

The repository includes `retired-vm-sizes.json`, but you can regenerate it from the Microsoft Learn source page:

```powershell
.\Update-RetiredAzVmSizesFile.ps1 -OutputPath .\retired-vm-sizes.json
```

The generated file contains retired or retirement-announced VM families and individual SKUs. Pass it to the main script with `-RetiredSizesPath` to use the latest data.

## Notes

- The script uses `Get-AzComputeResourceSku` and temporarily switches the current Az context to the requested subscription. It restores the original Az context before exiting.
- NVMe filtering is strict: a SKU must explicitly advertise `NVMe` in `DiskControllerTypes`.
- SCSI filtering includes SKUs that explicitly advertise `SCSI` and older Gen1-capable SKUs where `DiskControllerTypes` is missing.
- Local temporary/resource disk presence is derived from `MaxResourceVolumeMB`; values greater than zero are treated as having a local temporary/resource disk.
